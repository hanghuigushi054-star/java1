## 確認用
* **void型:** 戻り値がない（何も返さない） ex.System.out.println()
* **int型:** 整数値を返す ex.return 5;
* メソッド呼び出しはメソッドの 内側 に書く必要がある
* クラスレベルでは実行文を書けない
## 間違えやすいもの
* switch文のbreak ← バグの温床
* if文の波括弧 {} ← 論理エラーの原因
* Scanner.close() ← リソースリーク
* インデント統一 ← 可読性向上
* 変数名の改善 ← 保守性向上
* コンストラクタの引数型とフィールド型は必ず一致させる
* 内部クラスと static の関係
* コメントと実装の混同
* 配列に親クラス型を使う
* オーバーライドのうっかり忘れ

# 間違いやすいポイント
* == vs equals() - 参照比較と内容比較を区別する
* null チェック - NullPointerException 対策
* static vs instance - staticコンテキストから非staticにアクセスできない
* 配列は固定長 - 動的に増やす場合は ArrayList
* ループ変数のスコープ - ループ外では使えない
* String は不変 - メソッドの戻り値を代入しないと反映されない
* 変数の初期化 - 初期化なしで使うとエラー
* キャスト前に確認 - ClassCastException対策に instanceof を使う
* 戻り値を無視しない - メソッドの結果を活用する
* 一般的な Exception ではなく具体的な例外 をキャッチする
* 内部クラスは static にする - main()から使う場合
* コメントではなく実装する - 処理を書く
